# Typeless Mac 使用示例

## 场景 1: 快速记录会议笔记

**操作**:
1. 打开笔记应用（如 Apple Notes、Obsidian、VS Code）
2. 按 `Cmd+Shift+Space` 开始录音
3. 口述内容："嗯，今天那个会议的主要内容是，就是，我们需要在下周五之前完成项目的第一阶段，不对，应该是下周四，然后安排测试"
4. 停顿 2 秒或再按快捷键

**输出**:
```
今天会议的主要内容是：我们需要在下周四之前完成项目的第一阶段，然后安排测试。
```

## 场景 3: 编写代码注释

**操作**:
1. 在代码编辑器中，定位到需要注释的位置
2. 输入注释符号 `# ` 或 `// `
3. 按 `Cmd+Shift+Space`
4. 口述："这个函数的作用是，嗯，就是检查用户输入是否有效，如果无效的话就返回错误信息"
5. 停顿

**输出**:
```python
# 这个函数的作用是检查用户输入是否有效，如果无效则返回错误信息。
```

## 提示和技巧

### 1. 清晰发音
- **好**: 说话速度适中，发音清晰
- **差**: 过快或含糊不清

### 2. 适当停顿
- 句子之间稍作停顿
- 重要内容可以重复说明

### 3. 改口修正
- 说错了直接改口："不对，应该是..."
- LLM 会自动保留最后的正确版本

### 4. 结构化内容
- 使用 "第一、第二、第三" 来列举
- 说 "换行" 或停顿来分段

### 5. 专业术语
- 技术术语和英文词汇可以直接说
- 模型会自动识别混合语言

### 6. 快捷键技巧
- **单击**: 开始/停止录音（手动控制）
- **长录音**: 说完后停顿 2 秒自动结束
- **取消**: 录音后立即再按一次（音频太短会被忽略）

## 常见场景对比

| 场景 | 传统打字 | Typeless Mac |
|------|----------|--------------|
| 长段文字 | 慢，容易打错 | 快，自动纠错 |
| 移动办公 | 需要键盘 | 只需麦克风 |
| 多任务 | 需要看屏幕 | 可以边做边说 |
| 改口重写 | 手动删除 | 自动修正 |
| 中英混合 | 需要切换输入法 | 自动识别 |

---

## 最佳实践

### ✅ 适合使用 Typeless 的场景
- 长文本输入（邮件、报告、笔记）
- 快速记录（会议、灵感、待办）
- 移动办公（没有键盘）
- 多任务时（边做边说）

### ❌ 不适合的场景
- 需要精确格式的代码
- 包含大量特殊符号
- 非常嘈杂的环境
- 需要保密的内容（使用离线模式）

## 效果对比示例

### 示例 1: 改口修正

**原始语音**:
> "我今天要去，不对，我明天要去公司开会，然后，就是，准备一下材料"

**传统 ASR** (仅识别):
> "我今天要去，不对，我明天要去公司开会，然后，就是，准备一下材料"

**Typeless** (AI 润色):
> "我明天要去公司开会，准备一下材料。"

---

**传统 ASR**:
> "嗯，那个，我觉得这个方案，就是，还挺好的，嗯，可以试试"

**Typeless**:
> "我觉得这个方案还挺好的，可以试试。"

> 1. 完成报告

## 反馈与改进

使用过程中如果遇到识别或润色效果不理想，可以：

1. **调整模型**: 尝试更大的 ASR 模型（base 或 small）
2. **优化提示词**: 在 `config.yaml` 中调整 `system_prompt`
3. **切换 LLM**: 尝试不同的模型
4. **改善环境**: 使用更好的麦克风，减少背景噪音
